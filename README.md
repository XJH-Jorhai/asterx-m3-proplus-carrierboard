# AsteRx-m3 ProPlus Carrierboard

This repository contains the design boundary, component selection review, and implementation notes for an AsteRx-m3 ProPlus OEM receiver carrier board.

## Contents

- `AsteRx-m3_ProPlus_底板设计规范.md` - carrier board design specification.
- `AsteRx-m3_ProPlus_器件选型评审_定稿.md` - final component selection review.
- `AsteRx-m3_ProPlus_器件选型清单_定稿.csv` - final component selection table.
- `asterx_m3_proplus_carrier_feature_boundary.csv` - feature boundary and design scope table.
- `AGENTS.md` - project-specific handoff rules for future AI/EDA work.

## Current Hardware Direction

- AsteRx-m3 ProPlus OEM module on a custom carrier board.
- USB-C device port with optional 5V power input.
- DC 12V input, 24V tolerant front end.
- DC input protection with `TPS26631RGER`; `TPS26633RGER` is not the project main choice.
- USB-C VBUS protection with `TPS259472LRPWR`, then `TPS2121RUXR` power mux.
- 5V system rail generated by `LM76002RNPT` synchronous buck.
- 3.3V low-noise rail for AsteRx module generated by `TPS7A8300RGRR`.
- Antenna feed currently generated as fixed `4.2V_ANT`.
- RS232/RS485 input-side power generated as `3.3V_232485`.
- Ethernet through 10/100M RMII PHY and `HR911130A / C54408` LED MagJack; no PoE.
- MicroSD logging socket.
- COM1 isolated RS232 via `RSM232`, COM2 automatic-direction isolated RS485 via `TD301D485H-A`, both on one high-density D-Sub connector.
- COM3/COM4 LVTTL on GH connectors.
- MAIN and AUX1 antennas via MMCX-to-SMA pigtails.
- PPS, Event, REF IN/OUT external timing interfaces.
- SMA/USB/DC interface area is the only CHASSIS-PGND hard-bond area.
- USB Shield connects to CHASSIS; TNC antenna shield remains separated from internal I/O_GND/PGND.
- Separate button/interface board connected by FPC.

## EasyEDA Handoff

- Do not use schematic net ports. Use wires plus net labels; only GND may use the ground symbol.
- Mark NC pins with no-connect markers, not net labels.
- Use a 0.05 inch grid and keep IC support passives next to the IC.
- Use Chinese schematic comments and page notes.

Vendor manuals are intentionally ignored by git. Local copies may exist in the working directory; refer to official Septentrio documentation for authoritative module specifications.
